## Overview

This is a comprehensive benchmark dataset for research on **inclusive medical-health-care integrated service composition**, constructed using a bilateral resource integration method. Data is sourced from the [Hugging Face](https://huggingface.co/) open-source model community, covering 17,535 callable model services with full metadata, Quality of Service (QoS) measurements, 10,000 service composition workflows, corresponding natural language user requirements, and optimal service solutions generated by optimization algorithms.

**Collection Period:** March 2024 – September 2024

**Functional Coverage:** 18 top-level categories / 339 sub-categories (Text Generation, Image Classification, Object Detection, Automatic Speech Recognition, Text-to-Speech, and more)

**Deployment Regions:** North America, Europe, Asia-Pacific

## Repository Structure

```
📦 dataset/
├── raw_models.json          # Raw service metadata and QoS test data (18.8 MB)
├── normalized_model.json    # Normalized QoS data (6.54 MB)
├── requirements.json        # Natural language user requirement descriptions (11.1 MB)
├── workflow.json            # Service composition workflows (6.68 MB)
└── best_solution.json       # Optimal service solutions (0.1 MB)
```

Files are linked via **service name** and **array index position**:

```
raw_models.json  ←────────────────────  normalized_model.json
                                                 │
                                 ┌───────────────┘
                                 ▼
requirements.json[i]  ←──  workflow.json[i]  ──►  best_solution.json[i]
```

> The `i`-th requirement, `i`-th workflow, and `i`-th best solution correspond to each other by **array index**.

## File Descriptions

### 1. `raw_models.json` — Raw Service Data

**Structure:** JSON Object — keys are full service names (`author/model-name`), values are arrays of length 8.

| Index | Type | Field | Description |
|:---:|---|---|---|
| 0 | `String` | Functional description | Model introduction, training data, performance metrics, etc. (200–1000 chars) |
| 1 | `String` | Output note | Fixed text: `"the output, details can be found in the raw file."` |
| 2 | `Array[Float]` | Response / loading time series | Multi-call timing records in seconds (typically 5–10 entries) |
| 3 | `Integer/Float` | Downloads | Raw download count |
| 4 | `Float` | Likes | Raw like count |
| 5 | `Float` | Composite quality score | Range `[0, 1]`, computed from multiple QoS metrics |
| 6 | `Integer/Array` | HTTP status code(s) | Single code or list recording each call's response status |
| 7 | `String` | Function category path | Format: `"top-level/sub-level"`, e.g. `"audio-classification/emotion-recognition"` |

**HTTP Status Codes:**

| Code | Meaning |
|:---:|---|
| 200 | Success |
| 400 | Bad request |
| 403 | Forbidden |
| 404 | Not found |
| 429 | Rate limit exceeded |
| 500 | Internal server error |
| 503 | Service unavailable (load timeout) |

<details>
<summary>Example</summary>

```json
{
  "MIT/ast-finetuned-audioset-10-10-0.4593": [
    "Audio Spectrogram Transformer (AST) model fine-tuned on AudioSet...",
    "the output, details can be found in the raw file.",
    [24.63, 1.36, 1.79, 1.45, 1.52],
    128000,
    312,
    0.87,
    [200, 200, 200, 200, 200],
    "audio-classification/audio-event-classification"
  ]
}
```
</details>

### 2. `normalized_model.json` — Normalized QoS Data

**Structure:** JSON Array — each element is a service object with normalized QoS metrics.

> ⚠️ **Note:** The `service_name` field contains only the model name **without** the author prefix, unlike the keys in `raw_models.json`.

| Field | Type | Description | Range |
|---|---|---|:---:|
| `service_name` | `String` | Model name (no author prefix) | — |
| `function` | `String` | Function category path, e.g. `"audio-classification/emotion-recognition"` | — |
| `normalized_downloads` | `Float` | Normalized download count | `[0, 1]` |
| `normalized_likes` | `Float` | Normalized like count | `[0, 1]` |
| `normalized_response_times` | `Float` | Normalized average response time (lower = faster) | `[0, 1]` |
| `normalized_waiting_times` | `Float` | Normalized average loading time (lower = faster) | `[0, 1]` |
| `reliabilities` | `Float` | Reliability = HTTP 200 count / total requests | `[0, 1]` |
| `successabilities` | `Float` | Success rate = valid and correct outputs / total requests | `[0, 1]` |

All QoS metrics are normalized using **Min-Max normalization**:

$$\text{normalized\_value} = \frac{\text{value} - \min}{\max - \min}$$

<details>
<summary>Example</summary>

```json
{
  "service_name": "ast-finetuned-audioset-10-10-0.4593",
  "function": "audio-classification/audio-event-classification",
  "normalized_downloads": 0.744,
  "normalized_likes": 0.514,
  "normalized_response_times": 0.757,
  "normalized_waiting_times": 1.0,
  "reliabilities": 0.53,
  "successabilities": 0.93
}
```
</details>

### 3. `requirements.json` — User Requirement Descriptions

**Structure:** JSON Array — each element is a natural language requirement description (200–500 words).

Each requirement simulates a real-world business scenario and typically includes:

- **Business context:** Application scenario and target users
- **Functional needs:** Required service capabilities (e.g., image classification, speech recognition, TTS)
- **Performance constraints:** Specific QoS thresholds (e.g., response time < 3.0 s)
- **Optimization goals:** Metrics to optimize (e.g., minimize waiting time, maximize downloads)
- **Business objectives:** Expected outcomes (e.g., user engagement, satisfaction)

<details>
<summary>Example</summary>

```json
[
  "In order to enhance our multimedia application and cater to a diverse user base, we need a robust feature set that includes advanced classification capabilities for images, along with seamless speech ...",
  "..."
]
```
</details>

### 4. `workflow.json` — Service Composition Workflows

**Structure:** JSON Array — each element defines a service composition workflow and its optimization parameters.

| Field | Type | Description |
|---|---|---|
| `flow` | `Array[Array[String]]` | Workflow sequence: outer array = serial stages; inner array = candidate service categories for that stage |
| `obj_func` | `Object` | Objective function — keys are QoS metric names, values are weights |
| `cons` | `Object` | Constraints — keys are QoS metric names, values are thresholds |

**Optimization directions for `obj_func`:**

| Metrics | Direction |
|---|:---:|
| `normalized_downloads`, `normalized_likes`, `reliability`, `successability` | Maximize ↑ |
| `normalized_response_times`, `normalized_waiting_times` | Minimize ↓ |

**Constraint operators for `cons`:**
- Downloads, likes, reliability, successability: `> threshold`
- Response time, waiting time: `< threshold` (threshold = 0 means no constraint)

<details>
<summary>Example</summary>

```json
{
  "flow": [
    ["automatic-speech-recognition/multilingual-speech-recognition"],
    ["summarization/multilingual-text-summarization"]
  ],
  "obj_func": {
    "average number of downloads": 0.3,
    "average number of likes": 0.5,
    "average response time": 0.0,
    "average waiting time": 0.1,
    "reliability": 0.0,
    "successability": 0.1
  },
  "cons": {
    "average number of downloads": 0.0,
    "average number of likes": 0.0,
    "average response time": 0.0,
    "average waiting time": 0.0,
    "reliability": 0.0,
    "successability": 0.0
  }
}
```
</details>

### 5. `best_solution.json` — Optimal Service Solutions

**Structure:** JSON Array — each element is the optimal service configuration computed by an optimization algorithm.

| Field | Type | Description |
|---|---|---|
| `req_number` | `Integer` | Requirement index (0-based), corresponding to `requirements.json` and `workflow.json` |
| `best_value` | `Float` | Composite objective function score of the best solution |
| `best_solution` | `Array[Integer]` | Optimal solution: each integer is the **index into the candidate service list** for that workflow stage |
| `time` | `Float` | Algorithm solving time in seconds |

> **`best_solution` field:** The array length equals the number of stages in the corresponding `workflow.json[i].flow`. Each integer value is a positional index into the list of services in `normalized_model.json` that match the functional category of that stage.

**`best_value` formula example:**

$$\text{best\_value} = w_1 \cdot (1 - \text{norm\_response\_time}) + w_2 \cdot (1 - \text{norm\_waiting\_time}) + w_3 \cdot \text{norm\_downloads} + w_4 \cdot \text{successability}$$

<details>
<summary>Example</summary>

```json
{
  "req_number": 1,
  "best_value": 1.0234244839142772,
  "best_solution": [1, 1, 2],
  "time": 8.886938333511353
}
```
</details>

## Quick Start

### Requirements

- **OS:** Windows 10/11, Linux (Ubuntu 20.04+), macOS 11+
- **Python:** 3.7+
- **RAM:** 4 GB recommended (full dataset load ~100 MB)
- **Disk:** At least 100 MB free

### Install Dependencies

```bash
pip install numpy pandas

# Recommended for analysis and visualization
pip install matplotlib seaborn scikit-learn
```

### Usage Example

```python
import json
import pandas as pd

# 1. Load raw service data
with open('raw_models.json', 'r', encoding='utf-8') as f:
    raw_models = json.load(f)

service_name = "MIT/ast-finetuned-audioset-10-10-0.4593"
service = raw_models[service_name]
print(f"Category: {service[7]}")
print(f"Avg response time: {sum(service[2]) / len(service[2]):.2f}s")

# 2. Load normalized data
with open('normalized_model.json', 'r', encoding='utf-8') as f:
    normalized_models = json.load(f)

df = pd.DataFrame(normalized_models)
print(f"Total services: {len(df)}")
print(df.head())

# 3. Load requirements
with open('requirements.json', 'r', encoding='utf-8') as f:
    requirements = json.load(f)
print(f"Total requirements: {len(requirements)}")
print(f"First requirement: {requirements[0][:100]}...")

# 4. Load workflows
with open('workflow.json', 'r', encoding='utf-8') as f:
    workflows = json.load(f)
wf = workflows[0]
print(f"Workflow stages: {len(wf['flow'])}")
print(f"Objective weights: {wf['obj_func']}")
print(f"Constraints: {wf['cons']}")

# 5. Load best solutions
with open('best_solution.json', 'r', encoding='utf-8') as f:
    solutions = json.load(f)
sol = solutions[0]
print(f"Requirement index: {sol['req_number']}")
print(f"Best score: {sol['best_value']:.4f}")
print(f"Best solution (stage candidate indices): {sol['best_solution']}")
print(f"Solve time: {sol['time']:.2f}s")
```

## QoS Metrics

| Metric | Computation |
|---|---|
| **Response time** | Multiple independent calls (typically 10, over 1–2 weeks), recorded in seconds |
| **Reliability** | HTTP 200 count / total requests |
| **Successability** | Valid and correct responses / total requests |
| **Normalization** | All QoS metrics use Min-Max normalization, resulting in range `[0, 1]` |
